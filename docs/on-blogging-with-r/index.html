On blogging with R

Dec 30, 2017 #R #R Markdown #plot

As noted previously, I use R for school and work when doing statistical computing. I tend to prefer sharing nicely formatted HTMLs generated from analyses conducted in R using R Markdown. I was excited to learn that it was now possible to integrate “knitted” HTML output from an R Markdown (.Rmd) file to my Hugo blog with the assistance of the new R package, blogdown.

Quick start

blogdown states that the “main [blog platform] we support in blogdown is Hugo”. I had previously set up Hugo for building this website before learning of the compatibility with R Markdown through blogdown. Thus, after installing the blogdown package using install.packages("blogdown"), I did not need to further run blogdown::new_site() to generate a new site.

I then searched around for an example of an R Markdown file used in an active blog, and although I didn’t find one that quite met my needs, I did find some useful internet posts on the topic of using R to generate blogposts here, here, here, and here.

After setting up an R Markdown file for this blog post, I rendered the blog using blogdown::serve_site("~/Users/analagossai/u_blog/content/post/my_third_post.Rmd") typed into an R command line or program window.

Debugging

As usual, my main problems during setup was relative URLs. Namely, I was interested in displaying HTML widgets and had trouble finding them to render in my Hugo blog.

See below for a short example of my first blog post generated with blogdown and R Markdown!

sourceDir <- function(path, 
                      trace = TRUE, ...) {
    
    
    for (nm in list.files(path, pattern = "\\.[RrSsQq]$")) {
       if (trace) {
           source(file.path(path, nm), ...)
       }
    }
}
